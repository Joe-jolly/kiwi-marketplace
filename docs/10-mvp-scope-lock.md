# Kiwi Marketplace

## MVP Scope Lock

Version: 1.0

Status: Approved

---

# 1. Purpose

This document defines exactly what belongs inside the MVP.

Anything not explicitly listed is considered out of scope.

The purpose of this document is to prevent feature creep.

---

# 2. MVP Objective

Launch a working marketplace for migrants living in South Korea.

Core Flow:

Browse

↓

Search

↓

Open Listing

↓

Chat

↓

Reserve

↓

Call

↓

Deal

The MVP succeeds when this flow works reliably.

---

# 3. MVP Success Definition

The MVP is considered complete when a user can:

* Register
* Verify SMS
* Login
* Create Posts
* Upload Images
* Browse Listings
* Search Listings
* Save Favorites
* Chat
* Reserve Listings
* Access Phone Numbers
* Report Abuse

Without critical issues.

# 22. Feature Request Constitution

When a new feature is proposed:

Ask:

Does it help users move from:

Post

↓

Chat

↓

Deal

faster?

If not:

Reject for MVP.

Move to Future Roadmap.

---

# 23. Founder Protection Rule

Founders are especially vulnerable to feature creep.

Before approving any new feature:

Wait 24 hours.

Review business value.

Review implementation cost.

Review MVP impact.

Only then decide.

---

# 24. Scope Change Constitution

Changing MVP scope requires:

* Documentation Update
* Business Justification
* Explicit Approval

Scope changes must never happen casually.

---

# 25. Final Rule

MVP completion is more important than MVP perfection.

A launched marketplace with limited features is more valuable than a perfect marketplace that never launches.
